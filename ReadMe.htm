Prerequisites

  1. IIS (with ASP.NET)
  2. SQL Server 2008 (installed as the default instance)
  3. VS2010
  4. Silverlight 4 Tools for Visual Studio
  5. Silverlight 4 Toolkit
  6. Subversion Client

Build and Run Steps:

  1. Confirm that all the above tools are installed and configured correctly. If you can't figure out how to do that, maybe you don't need to be reading this :-).
  2. Check out the appropriate directory structure from subversion (http://alanta.unfuddle.com/svn/alanta_alanta/trunk/AlantaClient).
  3. Copy .\Alanta.Web\ClientAccessPolicy.xml to the root of your default IIS instance (e.g., "c:\inetpub\wwwroot"), and confirm that http://localhost/ClientAccessPolicy.xml is accessible.
  4. Add an entry for "127.0.0.1 dev.alanta.com" to your c:\windows\system32\drivers\etc\hosts file.
  5. Create the Alanta database through the SQL Server Management Studio tool.
  6. Open the.\AlantaClient.sln file in VS2010.
  7. Set the database deployment properties in the AlantaDB project by right-clicking the AlantaDB project, choosing "Properties" and then the "Deploy" tab.
    1. Configure deployment settings for: -> My isolated development environment
    2. Deploy action: -> Create a deployment script and deploy to the database
    3. Next to Target connection, click the "Edit" button:
      1. Server name: (local)
      2. Use SQL Server Authentication (with user name 'sa')
      3. Connect to the database "Alanta"
      4. Click on "OK".
      5. The resulting connection string should be:  Data Source=(local);Integrated Security=True;Pooling=False;MultipleActiveResultSets=True
    4. Target database name: -> Alanta
  8. Populate the Alanta database by right-clicking the AlantaDB project and choosing "Deploy".
  9. Set the startup projects by right-clicking the solution and choosing "Set startup projects".  The action for "Alanta.ConsoleHost" and the "Alanta.Web" projects should be set to to "Start"; all others should be set to "None".
  10. Set the startup page by navigating to Alanta.Web\Test\Test.aspx.  Right-click on that file and choose "Set as startup page."
  11. Set the web debugging options by right-clicking on the Alanta.Web project and choosing "Properties".  Select the "Web" tab and choose "ASP.NET" and "Silverlight" as the two active debuggers.
  12. Optional: build and launch the media server (http://alanta.unfuddle.com/svn/alanta_alanta/trunk/VoiceModule). This isn't strictly necessary, but the media features won't be available if it's not running.
  13. Optional: build and launch the RFB reflector server (http://alanta.unfuddle.com/svn/alanta_alanta/trunk/AlantaRfb). This isn't strictly necessary, but the desktop sharing features won't be available if it's not running.
  14. Optional: build and launch the file conversion server (http://alanta.unfuddle.com/svn/alanta_alanta/trunk/ConversionManager). This isn't strictly necessary, but the file sharing features won't be available if it's not running.
  15. Launch the project by hitting F5.
  16. In the web page that opens, click the button "Create Room as Guest".  The Alanta Client should load.
